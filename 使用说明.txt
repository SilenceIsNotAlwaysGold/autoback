抖音自动回复 - Windows 版（快速上手）
=====================================

【系统要求】
- Windows 10 / 11 64-bit
- 使用抖音、代理测试和 AI 功能时需要联网
- 端口 8020 空闲（被占会启动失败）
- 程序目录路径建议不带中文/空格

无需安装 Python、Node、Chrome、Playwright，也不需要管理员权限。
Chromium 浏览器内核已包含在分发包中，首次启动不会下载内核。


【三步上手】

1. 拷贝 + 解压
   - 把 dy_auto_reply_windows.zip 拷到目标电脑（U盘 / 网盘 / 局域网均可）
   - 解压到任意位置，例如 C:\dy_auto_reply\
   - 不要把 dy_auto_reply.exe 单独抽出来，它依赖同目录下的 _internal\

2. 启动程序
   - 双击 dy_auto_reply.exe
   - 看不到窗口是正常的（GUI 模式无控制台）
   - 通常几秒内浏览器会自动打开 http://127.0.0.1:8020

3. 配置 + 运行
   - 「账号管理」Tab -> 添加账号 + 填代理 URL -> 扫码登录
   - 「关键词规则」Tab -> 编辑回复规则 / AI 参数
   - 顶栏「启动主脚本」开始自动回复

详细教程见同目录的 教程-Windows.md


【数据位置】
所有用户数据均在用户的 Roaming 目录下：
%APPDATA%\dy_auto_reply\
   config\dy_reply.yaml         配置文件
   data\dy_reply.db             回复历史 / 状态
   data\browser_profiles\       各账号登录态
   logs\dy_main.log             主程序运行日志
   logs\launcher.log            启动器日志（排查闪退看这个）

Chromium 浏览器内核位于程序目录的 _internal\playwright-browsers\ 中。


【常见问题排查】

* 双击 exe 没反应 / 闪退
  → 看 %APPDATA%\dy_auto_reply\logs\launcher.log

* 浏览器没自动打开
  → 手动访问 http://127.0.0.1:8020

* 日志出现 "downloading Chromium"
  → 当前分发包不完整或仍是旧版本。重新下载完整 ZIP，并确认
    _internal\playwright-browsers\chromium-*\chrome-win64\chrome.exe 存在。

* 端口 8020 被占用
  → 命令行执行：netstat -ano | findstr 8020
    找到占用进程后关掉

* 杀毒软件误报为木马
  → PyInstaller 打包的 exe 经常被 Defender / 360 误报。
    把 dy_auto_reply.exe 加白名单即可。
    Windows SmartScreen 拦截：右键 -> 属性 -> 勾选「解除锁定」。


【清空 / 重置】
直接删除以下目录，下次启动会自动重建：
- 程序目录          C:\dy_auto_reply\
- 数据目录          %APPDATA%\dy_auto_reply\
