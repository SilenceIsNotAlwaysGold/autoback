# 抖音自动回复 — Windows 使用教程

> 一个软件管理多个抖音账号，自动回复私信和评论。每个账号走各自的代理 IP，互不关联。

---

## 一、安装

1. **解压** `dy_auto_reply.zip`，得到一个 `dy_auto_reply` 文件夹。
2. **把整个文件夹放到一个独立目录**，例如 `D:\dy_auto_reply\` 或 `C:\dy_auto_reply\`。
   - ⚠️ **不要**只复制 `dy_auto_reply.exe` 单个文件——它必须和旁边的 `_internal` 文件夹放在一起才能运行。
   - ⚠️ **不要**放在微信/QQ 的缓存目录、临时目录、或带中文特殊符号的路径里运行，容易出错。

---

## 二、首次启动

### 步骤 1：放行 Windows 安全拦截

软件没做数字签名，第一次打开会被 Windows 拦下，这是正常的。

- 双击 `dy_auto_reply.exe`，若弹出蓝色「**Windows 已保护你的电脑**」：
  点「**更多信息**」 → 出现「**仍要运行**」按钮 → 点它。
- 不需要管理员权限，普通双击运行即可。
- 如果被杀毒软件（360、火绒、电脑管家等）拦截删除：把 `dy_auto_reply.exe` 加入**白名单/信任区**，或临时关闭杀毒后再启动。
  （PyInstaller 打包的程序常被误报病毒，属正常现象。）

### 步骤 2：等待配置页面打开

分发包已经内置 Python、Playwright 和 Chromium 浏览器内核，**不会在目标电脑下载内核**。

- 首次启动只会在 `%APPDATA%\dy_auto_reply\` 创建配置和数据目录。
- 通常几秒内会自动打开配置页面；软件本身不依赖本机 Python、Node、Chrome 或 Playwright。
- 使用抖音、测试代理和调用 AI 接口仍然需要正常联网。

启动完成后，会**自动用默认浏览器打开配置页面**：

```
http://127.0.0.1:8020
```

如果浏览器没自动弹出，手动复制这个地址到浏览器打开即可。

> 💡 必须保留 `dy_auto_reply.exe` 和旁边完整的 `_internal` 文件夹。

---

## 三、配置代理（防止多账号被关联，强烈建议每个账号配）

多个账号如果用同一个本机 IP 登录，容易被抖音风控判定为关联号。**给每个账号配一个独立代理 IP** 是最重要的防护。

### 直接粘贴购买的代理（新功能，无需手动转格式）

在账号的「**代理 URL**」输入框里，**直接粘贴你买到的代理**即可，常见格式都支持：

```
182.40.197.72|14081|用户名|密码|到期日
```

也支持：
```
182.40.197.72|14081|用户名|密码
1.2.3.4:8080:用户名:密码
socks5://用户名:密码@1.2.3.4:8080
http://用户名:密码@1.2.3.4:8080
```

粘贴后，有两种方式整理：

1. **鼠标点一下输入框外面（失焦）** → 自动清洗成标准格式，例如
   `socks5://用户名:密码@182.40.197.72:14081`
2. 或点「**✨ 整理格式**」按钮 → 同样即时整理（不联网，秒回）。

> 到期日那一段会被自动忽略，账号密码里的特殊符号也会自动处理。

### 测试代理是否能用

点「**🧪 测试连通**」按钮：
- 软件会**自动联网识别**这是 socks5 还是 http 代理，并显示**出口 IP** 和延迟。
- 看到 ✅ + 出口 IP，说明代理可用。
- 看到 ❌，说明这个代理连不通，换一个或联系代理商。

> 🔒 **安全保障**：如果配了代理但代理连不通，软件会**直接中止**，绝不会偷偷用你的本机 IP 上号，避免账号被关联。

## 四、配置账号 + 扫码登录

在「**账号管理**」页：

1. 点新增账号，填：
   - **账号名**：自定义标识，如 `acc1`、`抖音小号1`（用于区分，别用特殊符号）。
   - **代理 URL**：按上面第三节粘贴 + 测试。
   - **登录超时**：默认 180 秒够用。
2. 点「**保存**」。
3. 点该账号的「**扫码登录**」按钮：
   - 稍等会弹出一个浏览器窗口，显示**抖音扫码二维码**。
   - 用**手机抖音 APP** 扫码确认登录。
   - 登录成功后，登录状态会自动保存（约 30 天有效），以后不用反复扫。

> 如果点了登录后窗口长时间黑屏或空白，先检查代理是否连通，再看第六节排查。

---

## 五、配置回复规则 + 启动

在「**关键词规则**」页：

- 顶部开关：
  - **关键词模式**：按关键词匹配回复。
  - **用户回复冷却**（默认 30 秒）：同一个人短时间内不重复轰炸。
- 添加规则：
  - **关键词**：用逗号分隔，可中英文，如 `多少钱,价格,怎么卖`。
  - **回复内容**：可写多行，每次随机选一条回，更自然。
  - 可设一条**默认回复**（没匹配到关键词时用）。
- （可选）开启 **AI 智能回复**：在配置里填好大模型接口，匹配不到关键词时由 AI 兜底回答。

设置好后，回到顶栏点「**启动主脚本**」，软件就开始自动盯私信/评论并回复了。

停止：点「**停止主脚本**」即可。

## 六、常见问题排查

**Q：点登录后弹出一个全黑的窗口？**
A：内核已经随程序打包，不是在下载。先确认代理测试通过，再更新显卡驱动或关闭可能拦截 Chromium 的安全软件后重试；程序已默认启用软件渲染以兼容远程桌面/VPS。

**Q：启动后浏览器没自动打开配置页？**
A：手动在浏览器打开 `http://127.0.0.1:8020`。

**Q：代理测试一直失败？**
A：① 确认代理没到期；② 换个格式重新粘贴再测；③ 让代理商确认账号密码、IP、端口正确；④ 有的代理只支持特定协议，软件会自动试 socks5/http，若都不通则是代理本身问题。

**Q：登录二维码刷不出来 / 一直转圈？**
A：多半是该账号的代理不通。先在账号里把代理「测试连通」通过，再点登录。

**Q：怎么确认登录真的走了代理、没暴露本机 IP？**
A：代理「测试连通」显示的出口 IP 就是登录会用的 IP；只要它不是你本机的公网 IP 即可。软件还做了 WebRTC 防泄漏，代理失败会自动中止。

**Q：想换电脑继续用？**
A：把 `%APPDATA%\dy_auto_reply\` 整个文件夹拷到新电脑的相同位置（含配置和登录态）。

## 七、数据和日志位置

在资源管理器地址栏输入 `%APPDATA%\dy_auto_reply` 回车即可打开：

```
%APPDATA%\dy_auto_reply\
  ├─ config\dy_reply.yaml        ← 账号 / 规则配置
  ├─ data\dy_reply.db            ← 回复记录（去重、每日上限）
  ├─ data\browser_profiles\      ← 各账号登录态
  └─ logs\
       ├─ launcher.log           ← 启动器日志
       ├─ login_账号名.log        ← 某账号登录过程日志（排查登录问题看它）
       └─ dy_main.log            ← 运行回复日志
```

> 遇到问题，把对应的 `login_账号名.log` 或 `dy_main.log` 发给技术支持，最容易定位。

---

## 八、卸载

1. 删除你解压的 `dy_auto_reply` 文件夹。
2. 删除 `%APPDATA%\dy_auto_reply\` 目录（配置和数据）。
3. 旧版本若曾下载过内核，可选删除 `%LOCALAPPDATA%\ms-playwright\`；当前自包含版本不使用它。
